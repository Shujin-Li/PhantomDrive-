基于 **PhantomDrive517 已有的代码**，从架构层面来对比：

---

## 架构差异对比

### 1. 整体架构分层

**DustRacing2D（扁平架构）：**
```
Game（单体类，186行头文件）
├── MCWorld（物理引擎）
├── Scene（渲染场景）
├── StateMachine（菜单→比赛状态机）
├── Race（比赛逻辑，766行）
│   ├── Car × 12
│   └── AI × 11
└── Renderer（OpenGL渲染）
```
所有东西都挂在 `Game` 类下面，`Race` 类直接管理车辆和比赛流程，物理引擎和比赛逻辑紧耦合。

**PhantomDrive517（分层架构）：**
```
GameEngine（统一引擎接口）
├── TrackManager（赛道管理）
├── VehicleSensor（数据采集）
├── CollisionDetector（碰撞检测）
├── GameViewWidget（渲染控件）
├── GameModeManager（游戏模式框架）
│   ├── ArcadeMode
│   └── LearningMode
├── TrafficObjectManager（交通对象管理）
├── PowerupManager（道具管理）
├── AIOpponentManager（AI管理）
└── ScoreManager（评分系统）
    └── TrafficRuleEnforcer
```
**核心差异：PhantomDrive 把物理引擎（MiniCore）和业务逻辑（GameMode）完全解耦了。** `GameEngine` 作为中间层，不直接依赖 `MCWorld`，而是通过 `VehicleSensor` 和 `CollisionDetector` 间接使用物理数据。

---

### 2. 物理引擎与游戏逻辑的分离

**DustRacing2D：**
- `Car` 类直接继承 `MCObject`，内置 `MCPhysicsComponent`
- 车辆加速/刹车/转向直接调用物理 API（`physicsComponent().addForce()`）
- 碰撞检测由 `MCWorld` 自动处理，`Car::collisionEvent()` 直接响应

**PhantomDrive517：**
- **没有 `Car` 类**，只有 `VehicleState` 数据结构
- 物理引擎（`MCPhysicsWorld`）被封装在 `src/physics/`，但**没有接入游戏循环**
- 车辆状态由外部通过 `GameEngine::updatePlayerVehicle()` 传入
- 碰撞检测由独立的 `CollisionDetector` 类处理，不是物理引擎自动触发

**结论：PhantomDrive 的设计是"数据驱动"——车辆位置/速度由外部计算后喂给引擎，而不是由物理引擎驱动车辆。**

---

### 3. 游戏模式框架（PhantomDrive 独有）

**DustRacing2D：**
- 只有 `Race` 类一种模式（12辆车跑圈）
- 通过 `Game::Mode` 枚举区分（OnePlayerRace / TwoPlayerRace / TimeTrial / Duel）
- 模式切换由 `StateMachine` 控制

**PhantomDrive517：**
- `GameMode` 抽象基类 + `GameModeManager` 管理器
- `ArcadeMode`（街机模式）和 `LearningMode`（学习模式）
- `ModeTransition` 处理模式切换
- **支持运行时动态切换模式**（通过信号槽）

```cpp
// PhantomDrive 的模式切换
GameModeManager::instance()->switchMode("learning");
emit modeChanged(newMode);
```

---

### 4. 交通对象系统（PhantomDrive 独有）

**DustRacing2D：**
- 赛道上只有装饰物（树、石头、看台）和 Bridge（桥）
- 没有交通信号灯、限速牌、行人区域

**PhantomDrive517：**
- `TrafficObject` 抽象基类
- `TrafficLightObject`（红绿灯，支持红/黄/绿状态切换）
- `SpeedLimitSignObject`（限速牌，带限速值）
- `PedestrianCrossingObject`（行人区域/斑马线）
- `TrafficObjectManager` 统一管理所有交通对象的状态和更新

---

### 5. 道具系统（PhantomDrive 独有）

**DustRacing2D：**
- 没有道具系统

**PhantomDrive517：**
- `Powerup` 抽象基类
- `PowerupBox`（道具盒，赛道上拾取）
- `PowerupManager` 管理道具生成/拾取/效果
- 支持多种道具类型（加速、护盾等）

---

### 6. 数据采集与评分系统（PhantomDrive 独有）

**DustRacing2D：**
- 只有简单的计时和排名（`Timing` 类）
- 成绩存储在 `Database`（SQLite）

**PhantomDrive517：**
- `DrivingDataCollector`（驾驶数据采集）
- `DrivingDataStorage`（数据存储）
- `DrivingScoreCalculator`（驾驶评分计算）
- `TrafficRuleEnforcer`（交通规则执行器，检测超速/闯红灯）
- `ScoreManager`（评分管理）
- `DrivingReportWidget`（数据报表 UI）

**这是 PhantomDrive 最大的特色——不仅仅是赛车游戏，还加入了"驾驶学习"的概念。**

---

### 7. 车辆传感器系统（PhantomDrive 独有）

**DustRacing2D：**
- 没有传感器概念，车辆状态直接读取

**PhantomDrive517：**
- `VehicleSensor` 类，模拟车辆传感器数据采集
- 采样间隔可配置（默认 50ms）
- 采集数据：位置、速度、加速度、角速度
- 通过信号槽异步输出数据（`sensorDataReady`）

---

### 8. 渲染架构差异

**DustRacing2D：**
```
MCWorld → MCWorldRenderer → MCGLScene → OpenGL
         └── 3D 透视相机
         └── GLSL Shader
         └── 粒子系统
```

**PhantomDrive517：**
```
GameEngine → GameViewWidget → QPainter
           └── 2D 正交相机
           └── RenderState 数据结构
           └── 纯色几何图形（占位）
```

---

### 9. 赛道系统差异

| 维度 | DustRacing2D | PhantomDrive517 |
|------|-------------|-----------------|
| **赛道格式** | `.trk` XML 文件（含 Tile 旋转/翻转） | `TrackData` 内存对象 |
| **Tile 类型** | 20+ 种（含 Corner45/Corner90/Bridge） | 9 种（基础类型） |
| **弯道拼接** | 自动旋转 Tile | 固定方向 |
| **Checkpoint** | `Route` / `TargetNode`（AI 导航用） | `Checkpoint` 类（简单坐标+半径） |
| **赛道数据库** | `TrackDatabase`（加载/缓存） | `TrackDatabase` + `TrackIO`（读写） |

---

### 10. AI 架构差异

**DustRacing2D：**
- `AI` 类（197行），直接控制 `Car` 的转向和加速

- 基于 `Route` / `TargetNode` 的路点导航
- PID 控制器转向

**PhantomDrive517：**
- `AIOpponent` 抽象接口
- `AIOpponentFSM`（有限状态机）
- `AggressiveAI` / `ConservativeAI` / `StrategicAI`（3种 AI 性格）
- `AIProfile`（AI 配置：速度/反应时间/激进程度）
- 通过 `GameEngine::onAIDecision()` 信号槽通信

---

### 总结

| 特性 | DustRacing2D | PhantomDrive517 |
|------|-------------|-----------------|
| **架构风格** | 单体扁平（Game 管一切） | 分层模块化（GameEngine → 各子系统） |
| **物理耦合** | 紧耦合（Car 继承 MCObject） | 松耦合（VehicleState 数据驱动） |
| **游戏模式** | 单一 Race 类 | GameMode 框架（可扩展） |
| **交通对象** | 无 | 红绿灯/限速牌/行人区域 |
| **道具系统** | 无 | PowerupManager |
| **数据采集** | 简单计时 | DrivingDataCollector + ScoreManager |
| **传感器** | 无 | VehicleSensor |
| **AI 架构** | 单一 AI 类 | FSM + 多种 AI 性格 |
| **渲染** | OpenGL 3D | QPainter 2D（占位） |

**PhantomDrive 的架构优势：模块化、可扩展、支持"学习模式"等新玩法。**
**PhantomDrive 的架构劣势：缺少真正的物理驱动，车辆只是数据壳。**
